### B.2 Finalizer Implementation
File: internal/lorethread/force_finalize.go (NEW)
- Type: Finalizer struct with all required dependencies
- Method: ForceFinalize(ctx, guildID, channelID, requesterUserID)
- Returns: (*ForceFinalizeResult, error)
- Error types:
  * ErrNoOpenSession: No open session in channel
  * ErrNotConversationStarter: Requester != session starter
- Pipeline: Same as worker.processSession
  1. Fetch open session with starter
  2. Authorize requester
  3. Check guild settings
  4. Check rate limit
  5. Fetch parent message
  6. Fetch recent messages
  7. Filter to lore messages
  8. Generate title + summary
  9. Create thread
  10. Store result + anchor
  11. Emit metrics

## BEHAVIOR CHANGES

- No way to finalize lore session early
- Any user could theoretically request thread creation (not implemented)

- Users can call lore_finalize_now to close session early
- ONLY conversation starter can finalize (enforced at tool level)
- Persona guides LLM to use tool correctly and refuse non-starters
- Honesty requirement: Never pretend to create thread if tool fails
