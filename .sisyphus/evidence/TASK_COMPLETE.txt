### Key Insight
**Bug Class**: "Capture gated on decision.Should silently swallowed all non-trigger lore chatter"

The capture call was placed AFTER the early return on `!decision.Should`, causing all non-trigger messages to skip capture entirely. Result: `lore_sessions` table remained empty despite active channel chatter.

**Fix**: Move capture BEFORE Router.Decide(). Capture now fires for all allowed-channel messages from real users, independent of reply decision. The adapter handles allow-list filtering independently.

- Capture still respects LoreCaptureTimeout configuration
- Bot messages still excluded (event.IsBot check)
- Non-blocking goroutine pattern preserved

The fix is complete, tested, and verified. Lore capture will now fire for all allowed-channel messages, enabling proper population of lore_sessions table and downstream lore thread creation.
