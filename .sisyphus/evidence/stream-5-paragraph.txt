## Implementation Details

- Paragraph boundary detection: `\n\n` in buffer AND buffer >= 200 bytes triggers flush
- Soft threshold: buffer >= 1600 bytes triggers SplitMessage with 1600 char limit
- Final flush: Flush() emits all remaining content via SplitMessage
- Per-instance serialization via mutex
- Idempotent Flush() after close

## Behavior Verified

1. Emits at paragraph boundaries when buffer exceeds 200 bytes
2. Flushes remainder on explicit Flush() call
3. Respects rate limits via ChannelRateLimiter
4. Handles long paragraphs > 2000 chars via SplitMessage
5. Silently drops pushes after close
