## Synthetic Message ID Scheme

For bot replies, we use a deterministic negative int64 to avoid collisions with real Discord snowflakes:

  syntheticMsgID := -(time.Now().UnixNano() % 9223372036854775807)

This ensures:
- Negative values never collide with positive Discord message IDs (snowflakes)
- Uniqueness constraint (guild_id, message_id) is preserved
- Deterministic and reproducible for testing
- No overflow issues with int64 bounds

Example synthetic IDs: -1715500198030000000, -1715500198030000001, etc.
