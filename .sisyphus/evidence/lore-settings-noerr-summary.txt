TASK: Fix ErrNoRows Handling in LoreGuildSettings
COMPLETED: 2026-05-14T07:38:53Z

SUMMARY
=======
Fixed repository layer to treat PostgreSQL "no rows in result set" as a valid state (default-OFF) 
rather than an error for lore feature queries. This prevents spurious error logs in the capturer 
when guilds don't have explicit settings rows.

CHANGES MADE
============

1. internal/repository/lore_guild_settings.go
   - Added imports: "errors" and "github.com/jackc/pgx/v5"
   - IsEnabled(): Now returns (false, nil) when pgx.ErrNoRows encountered
   - CountThreadsThisHour(): Now returns (0, nil) when pgx.ErrNoRows encountered
   - GetSettings(): Left unchanged (callers may want to distinguish "not configured")

2. internal/repository/lore_guild_settings_test.go
   - Added 5 new test cases:
     * IsEnabledMissingGuildReturnsDefaultFalse: Verifies (false, nil) for missing guild
     * IsEnabledWithEnabledTrue: Verifies (true, nil) when enabled=true row exists
     * IsEnabledWithEnabledFalse: Verifies (false, nil) when enabled=false row exists
     * CountThreadsThisHourMissingBucketReturnsZero: Verifies (0, nil) for missing bucket
     * CountThreadsThisHourAfterIncrement: Verifies (1, nil) after one increment

BEHAVIOR CHANGE
===============

BEFORE:
  - Guild without lore_guild_settings row → IsEnabled() returns (false, "failed to check if lore is enabled: no rows in result set")
  - Capturer logs: lore_settings_check_error guild=... error="failed to check if lore is enabled: no rows in result set"

AFTER:
  - Guild without lore_guild_settings row → IsEnabled() returns (false, nil)
  - Capturer does not log error; feature simply remains OFF until operator enables it
  - Correct semantics: feature OFF by default unless explicitly enabled
