## Implementation Summary

### schema.go Changes
Added two new methods to *Schema:

1. ToOpenAIFunction() map[string]interface{}
   - Converts schema to OpenAI function-calling format
   - Returns nested structure: {"type":"function","function":{...}}
   - Builds properties map with type and description for each field
   - Collects required field names into required slice
   - Maps Kind enums: string→"string", number→"number", bool→"boolean", object→"object", array→"array"
   - Falls back to "string" for unknown kinds

2. kindToOpenAIType(kind Kind) string
   - Helper method for Kind to OpenAI type string conversion
   - Handles all 5 Kind constants

### schema_test.go Changes
Added TestSchema_ToOpenAIFunction_Shape test:
- Tests schema with 1 required string field (query)
- Tests schema with 1 optional number field (limit)
- Tests schema with 1 optional bool field (verbose)
- Validates nested structure: type, function, parameters, properties, required
- Asserts correct type mappings: string→"string", number→"number", bool→"boolean"
- Verifies required array contains only required field names
