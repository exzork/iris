======================================

Date: 2026-05-12

## Summary

Fixed three critical bugs in the streaming pipeline that caused silent hangs:

1. StreamingSender.Push() silently swallowed errors including ErrRateLimitExceeded
2. SSE scanner blocked forever if upstream proxy stalled mid-stream
3. HTTP client timeout (30s) was shorter than orchestrator JobTimeout (45s)

## Key Improvements

1. Error Propagation:
   - StreamingSender.Push() now returns errors immediately
   - Orchestrator tracks errors and aborts stream on critical failures
   - Rate limit errors logged with visibility (streaming_sender_rate_limited)

2. Context Responsiveness:
   - SSE scanner uses goroutine + channel pattern
   - Respects ctx.Done() in real-time without blocking on scanner.Scan()
   - Cancellation detected within 1.5s even with stalled upstream

3. Timeout Handling:
   - ChatStream/ChatWithToolsStream wrapped with 120s timeout
   - Ensures hard ceiling even if caller forgets deadline
   - HTTP client timeout (30s) still used for non-streaming requests

4. Lock Refresh Logic:
   - Only refreshes conversation lock if at least one chunk was sent
   - Prevents opening active-conversation window on zero-chunk errors
   - Maintains lock for partial responses to enable follow-ups
